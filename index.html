Motivation: Why Joker?

In today's fast-paced digital world, moments of simple joy and spontaneous laughter are becoming rare. Existing entertainment devices are often passive or require complex interactions.

Joker aims to solve this by providing an interactive, tangible, and localized humor experience. It's not just a screen; it's a smart companion that reacts to physical inputs (keypad), understands context (ranking system), and breaks language barriers (real-time translation) to deliver happiness instantly.

Core Features

Joker integrates multiple IoT capabilities to create a rich user experience:

  • Smart Retrieval: Fetches random jokes via API based on user-selected categories (Programming, Dark, Pun, etc.).
  • Real-time Translation: Automatically translates English jokes into Korean for a bilingual experience.
  • Physical Interaction: Supports 'Keypad Input' for controls.
  • Auditory & Visual Feedback: Adaptive Buzzer sounds based on ratings.
  • Ranking System: Tracks user ratings in the cloud and displays the 'Top 3 Hall of Fame'.

Implementation & Workflow

The project utilizes a cloud-offloaded architecture to handle data processing, translation, and database management efficiently. The system retrieves different categories of jokes based on keypad input and processes real-time translations. The operational details are as follows:

1. Cloud-Offloaded Proxy Architecture

Fetching Joke Workflow

A Server-Side Proxy (Make.com) is utilized to handle heavy API requests and prevent memory overflow on the device. The fetching workflow is executed as follows:
1) Sending a specific category to the proxy from the device.
2) Fetching joke data from external APIs via the proxy.
3) Logging the retrieved jokes onto Google Sheets.
4) Translating the content into Korean using the integrated Google Translate function.
5) Returning optimized text data back to the ESP32.

2. Logging Rating & Automatic Ranking (App Script)

Rating Submission Workflow

Joke Database in Google Sheets

When a rating is submitted from the Wokwi simulator, the joke and its rating are stored in Google Sheets. To maintain a real-time leaderboard, Google App Script and Triggers are utilized to automatically sort the entries in descending order based on ratings.

Google App Script (Sorting Logic)

Automation Trigger Settings

The detailed Logging Task steps:
1) Sending the joke content and its user-assigned rating from the Wokwi simulator to the proxy.
2) Identifying the corresponding joke in Google Sheets and retrieving the row index.
3) Updating the row with the new rating, which triggers the automatic descending sort script.
4) Returning a success confirmation and updated ranking data to the device.

3. Cloud-Powered Bilingual Display & Ranking View

To support localization without local processing overhead, translation logic is embedded within the proxy. Additionally, the sorted ranking can be retrieved and displayed on the device through a dedicated menu.

Step 2: Fetching sorted Ranking from Cloud

Final display and ranking tasks:
1) The system displays the original English joke on the LCD while outputting the Korean translation via the Serial monitor.
2) Upon selecting the 'Rank' option (Key 'A') in Wokwi, the device sends a request to the proxy to fetch the top-rated jokes from the sorted Google Sheet.
3) The device receives the data and displays the fetched "Hall of Fame" ranking on the ILI9341 screen.
