Motivation: Why Joker?

In today's fast-paced digital world, moments of simple joy and spontaneous laughter are becoming rare. Existing entertainment devices are often passive or require complex interactions.

Joker aims to solve this by providing an interactive, tangible, and localized humor experience. It's not just a screen; it's a smart companion that reacts to physical inputs (keypad), understands context (ranking system), and breaks language barriers (real-time translation) to deliver happiness instantly.

Core Features

Joker integrates multiple IoT capabilities to create a rich user experience:

  • Smart Retrieval: Fetches random jokes via API based on user-selected categories (Programming, Dark, Pun, etc.).
  • Real-time Translation: Automatically translates English jokes into Korean for a bilingual experience.
  • Physical Interaction: Supports 'Keypad Input' for controls.
  • Auditory & Visual Feedback: Adaptive Buzzer sounds based on ratings.
  • Ranking System: Tracks user ratings in the cloud and displays the 'Top 3 Hall of Fame'.

My Task

I was responsible for developing the core interactive logic of Joker: the Dynamic Ranking System and the Real-time Status Timer. My goal was to ensure that user feedback is reflected in the cloud leaderboard instantly and that the device provides clear information about data freshness.

1. Cloud-Based Ranking System

Automated Leaderboard Management (Google Sheets & App Script)

The Ranking System is designed to create a "Hall of Fame" for the best jokes based on real-time user ratings (1-5):

  • Dynamic Sorting: I developed a Google App Script that automatically triggers whenever a new rating is submitted. It sorts the entire joke database in descending order based on scores.
  • Event-Driven Automation: By setting up an On-Edit Trigger, I eliminated manual database management, allowing the ESP32 to always fetch the most popular jokes for its "Ranking Mode".
  • Resilient Logic: To overcome the Wokwi simulator's HTTPS restrictions, I verified the sorting logic on the cloud-side to ensure the leaderboard is always ready for physical hardware deployment.

2. Status Timer & UI Optimization

V1: Elapsed Time (Relative)

V2: Final Status Timestamp (Absolute)

I iterated on the timer feature to improve both technical stability and user readability:

  • Technical Pivot: Initially, I implemented a feature showing elapsed seconds (V1). However, this caused performance overhead and synchronization conflicts with external APIs.
  • UI Optimization: To ensure system stability, I redesigned the feature to display a Static Update Timestamp (V2). This provides a clear, crash-free indicator of when the content was last refreshed, significantly improving overall UI reliability.
